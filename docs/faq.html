Help & FAQ

Find quick answers to common support questions regarding Burn to the Brim.

Resolved in v3.0.0

BTTB can't find WNASPI32.DLL or crashes on startup?

This is fully resolved in Burn to the Brim 3.0.0! The legacy Delphi branch (v2.9.0 and below) relied heavily on legacy ASPI interface drivers to interact with hardware CD writers, which frequently caused crashes on modern 64-bit Windows installations.

Our new **C++20 codebase** has retired the ASPI dependency completely. BTTB now runs natively, instantly, and with 100% stability on all modern versions of Windows 10/11 and Linux out-of-the-box.

BTTB states it fills a number of CDs, but results show one less?

BTTB *attempts* to fill the specified number of media, but it will end up selecting fewer if the remaining items cannot fit within your configured **Slack Space** threshold. You can easily adjust the Slack Capacity variable directly from the main solver settings to allow a larger or smaller packing tolerance.

Cannot find ISO creation tools on Windows?

To use the native **Create ISO Image** dialog on Windows, BTTB calls standard command-line tools `genisoimage` or `mkisofs`. If both are missing, BTTB will print detailed instructions in its dialog window explaining how to quickly install them using package managers like **Scoop** (`scoop install cdrtools`), **Chocolatey** (`choco install cdrtools`), **Cygwin**, or **MSYS2**.

I have a bug or question that isn't addressed here?
