製品説明

Burn to the Brim は、プロフェッショナルなファイルパッキングおよびストレージ最適化ユーティリティです。古典的なビンパッキング問題と部分和問題の最適化チャレンジを解決し、ファイルやフォルダを書き込み可能なCD、DVD、ブルーレイ、USBメモリ、またはカスタム容量の仮想ドライブなどのターゲットストレージメディアに、無駄なスペースをほぼゼロにして完璧に収めることができます。

高度に最適化された再帰的バックトラッキングソルバーを採用し、アプリケーションはディレクトリ構造とファイルサイズを分析して、ほぼ完璧なレイアウト構成をミリ秒単位で計算します。非ブロッキングのマルチスレッドエンジンにより、非常に大規模なデータセットのソリューションを計算している間でも、ユーザーインターフェースは完全に滑らかでインタラクティブな状態を維持します。

主な機能

  • →
    マルチボリュームスパンニング: シングルドライブの容量を超える膨大なファイル階層を、最適なサイズに分割して自動的にボリュームフォルダに格納します。
  • →
    Cauchy Reed-Solomon PAR3パリティ: エラー訂正用のPAR3パリティアーカイブを自動生成し、オーバーヘッドをレイアウトチェックに直接反映させます。
  • →
    整合性検証とデータ修復: パリティファイルと照合してボリュームをスキャン・検証し、破損したアイテムを特定して、複製による非破壊的な復元を実行します。
  • →
    Explorerコンテキストメニュー統合: Windows Explorerの右クリックメニューを介して、アクティブなセッションにファイルやフォルダを直接瞬時に追加します。
  • →
    ハイブリッドGUI/CLIエンジン: 軽量なネイティブUI（Win32 / GTK 4）と、スクリプト作成用の完全なコマンドラインモードをシームレスに切り替えます。
  • →
    Unicodeおよび長パスのサポート: 国際文字エンコーディングおよび最大32,767文字のWindows UNC長パスと完全に互換性があります。
  • →
    非ブロッキングツリー表示: メインループのアイドルサイクルを利用して結果をインクリメンタルに描画し、GUIのフリーズを防ぎます。
  • →
    小数点MB単位の容量精度: カスタムの180 KBや0.18 MBなど、正確なパッキングのために微細な容量入力と保存をサポートします。
  • →
    オフラインJSONインデックス作成: 出力時に標準化された index.json ディレクトリカタログを生成し、オフライン閲覧用にファイル属性をリストします。
  • →
    ルールベースと意味的グループ化: 標準のワイルドカード、正規表現、または文脈を考慮した意味的クラスタリングを使用して、ファイルを同じメディアにまとめます。

プライバシーとパフォーマンスの約束

Burn to the Brimは、ユーザーのプライバシー保護、システム性能の最大化、クリーンなソフトウェア基準を念頭に置いて構築されています：

  • ✗
    厳格なオフライン実行（スパイウェアなし）: アプリケーションは完全にローカルで動作します。ディレクトリパス、ファイル名、または個人情報をネットワーク経由で記録、追跡、送信することは一切ありません。
  • ✗
    アドウェアなし: アプリケーションに広告、商業バナー、またはスポンサーポップアップは含まれていません。
  • ✗
    超軽量フットプリント: 静的にコンパイルされたネイティブC++コードにより、メモリ使用量を最小限に抑え、フレームワークのオーバーヘッドをゼロにします。
  • ✗
    システムを汚さない設計: インストーラーはWindows統合に必要な設定のみを登録し、アプリケーションが不要なレジストリエントリを残すことはありません。
  • ✗
    クリーンなアンインストール: 標準のアンインストール手順により、すべてのプログラムディレクトリとショートカットがシステムから完全に削除されます。
